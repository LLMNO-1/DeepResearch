## 10.1 前端（`static/index.html`）

前端是一个**单文件 SPA**，不依赖任何前端框架（无 React/Vue），纯原生 HTML + CSS + JavaScript。

### 技术选型

- 无框架（原生 DOM 操作）
- 内嵌 CSS（CSS 变量体系）
- `fetch` API 调用后端
- 轮询模式追踪任务状态

### 交互流程

```
1. 首页表单 → 填写研究主题、目标、读者、范围
       ↓ POST /api/v1/research-projects
2. 轮询 GET /api/v1/tasks/{task_id}
       ↓ status: succeeded
3. GET /api/v1/research-projects/{id}/outline
       ↓ 展示大纲树
4. 用户确认或提交修改意见
       ↓ PUT /api/v1/research-projects/{id}/outline
5. POST /api/v1/research-projects/{id}/report-tasks
       ↓ 轮询等待完成
6. GET /api/v1/research-projects/{id}/reports/latest
       ↓ 展示 HTML 报告
```

前端不解析报告内容，直接通过 `iframe` 或 `innerHTML` 渲染后端返回的完整 HTML。

## 10.2 MongoDB 部署

项目提供了自定义的 MongoDB Docker 镜像（`mongo/Dockerfile.mongodb`），默认**无鉴权**。

```bash
# 构建
docker build -f mongo/Dockerfile.mongodb -t deep-research-mongo:7.0 mongo

# 运行（无鉴权本地开发）
docker run -d --name deep-research-mongo \
    -p 27017:27017 \
    -v deep-research-mongo-data:/data/db \
    deep-research-mongo:7.0
```

连接串：`mongodb://localhost:27017`，数据库名 `deep_research`。

### 数据集

MongoDB 中有三个集合（自动创建，无需手动建表）：

| 集合 | 存储内容 | 关键索引字段 |
|------|---------|------------|
| `research_projects` | 项目文档（包含大纲、章节、来源等所有数据） | `project_id` |
| `research_tasks` | 后台任务状态 | `task_id`, `project_id` |
| `report_versions` | 报告版本元数据（HTML 存对象存储） | `project_id`, `version` |

当前代码中没有显式创建索引。对生产环境建议在 `project_id` 和 `task_id` 上创建唯一索引。

## 10.3 RAGFlow 部署

`ragflow-docker/` 目录包含完整的 RAGFlow 本地部署方案。

### 服务编排

```yaml
# ragflow-docker/docker-compose.yml + docker-compose-base.yml
services:
  tei:           # 文本 Embedding 推理服务（BGE-M3 模型）
  mysql:         # RAGFlow 元数据存储
  es01:          # Elasticsearch 全文检索
  minio:         # 对象存储（文档/PDF 原文）
  redis:         # 缓存和任务队列
  ragflow-cpu:    # RAGFlow 主服务
```

### 关键配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| RAGFlow 镜像 | `swr.cn-north-4.myhuaweicloud.com/infiniflow/ragflow:v0.25.6` | 华为云国内镜像 |
| TEI 镜像 | `hotchpotch/tei-blackwell-testing` | Blackwell GPU 兼容版 |
| RAGFlow Web 端口 | `80` | 管理界面 |
| RAGFlow API 端口 | `9380` | 后端调用的检索 API |
| TEI 端口 | `6380` | Embedding 服务 |
| MySQL 端口 | `5455` | 避免与宿主机冲突 |

### 常用运维命令

```bash
# 启动全部服务
docker compose -f ragflow-docker/docker-compose.yml up -d

# 查看状态
docker compose -f ragflow-docker/docker-compose.yml ps

# 查看 RAGFlow 日志
docker compose -f ragflow-docker/docker-compose.yml logs -f ragflow-cpu

# 停止服务
docker compose -f ragflow-docker/docker-compose.yml stop

# 不要用 down -v！会删除 Docker volume（虽然当前用 bind mount，但仍需谨慎）
```

### 与本项目的集成

RAGFlow 是可选的——通过 `.env` 中的 `ENABLE_RAGFLOW` 开关控制：

```
ENABLE_RAGFLOW=true
RAGFLOW_BASE_URL=http://localhost:9380
RAGFLOW_API_KEY=your-ragflow-api-key
```

- 启用时：SearchAgent 获得 3 个工具（公开搜索 + 网页读取 + RAGFlow 检索）
- 未启用时：SearchAgent 只有 2 个工具（公开搜索 + 网页读取）

## 10.4 部署架构总览

```
┌─────────────────────────────────────────────────┐
│                    宿主机                        │
│                                                  │
│  ┌──────────┐  ┌────────────┐                   │
│  │ FastAPI  │  │ static/     │                   │
│  │ :8000    │  │ index.html  │                   │
│  └────┬─────┘  └────────────┘                   │
│       │                                          │
│       ├──→ MongoDB :27017 (Docker)               │
│       │      └── research_projects               │
│       │      └── research_tasks                  │
│       │      └── report_versions                 │
│       │                                          │
│       ├──→ Tavily API (云端)                     │
│       │                                          │
│       └──→ RAGFlow :9380 (Docker)  ←─可选       │
│              └── TEI :6380 (Embedding)           │
│              └── ES (全文检索)                   │
│              └── MinIO (文档存储)                │
│              └── MySQL (元数据)                  │
│              └── Redis (缓存)                    │
│                                                  │
│  ┌──────────────────────┐                       │
│  │ 报告 HTML 本地存储    │                       │
│  │ reports/ 目录         │                       │
│  └──────────────────────┘                       │
└─────────────────────────────────────────────────┘
```

### 最小可运行配置

只需启动两个服务：
1. **MongoDB Docker**（必须）
2. **FastAPI**（`uvicorn app.main:app`）

不需要 Tavily、不需要 RAGFlow。缺少这些时搜索工具返回 `skipped`，Agent 输出占位内容。

### 完整功能配置

需要额外启动和配置：
1. RAGFlow Docker Compose 全套服务
2. Tavily API Key（在 tavily.com 注册获取）
3. OpenAI 或 DeepSeek API Key

## 10.6 当前版本的局限性

| 方面 | 现状 | 影响 |
|------|------|------|
| 后台任务 | `asyncio.create_task`，无持久化 | 进程重启丢失排队中的任务 |
| 认证 | 无，CORS 全开 | 只适合内网或本地使用 |
| 前端 | 单文件，原生 JS | 功能有限，无复杂交互 |
| 测试 | 配置了 pytest 但无测试用例 | 重构风险较高 |
| 对象存储 | MinIO 是 stub | 报告 HTML 只能存本地 |
| Redis | 配置了但未使用 | 预留扩展点 |
| 并发 | 单进程 | 无法水平扩展 |
