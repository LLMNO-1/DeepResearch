## 9.1 核心设计：渲染不调 LLM

```
research_result (Agent 产出)  →  [确定性 Python 代码]  →  HTML 报告
                                     ↑
                              不调用 LLM、不搜索、不读网页
                              不新增事实、不修改结论
```

渲染引擎的约束在 `get_report_render_schema()` 中明确声明：

```python
"content_boundary": {
    "allowed": ["调整版式", "生成目录", "渲染引用脚注",
                "渲染表格", "渲染图表占位", "生成参考来源列表"],
    "forbidden": ["新增事实", "新增来源", "新增结论",
                  "改写证据链", "调用搜索工具"],
}
```

**为什么确定性渲染很重要**：如果在渲染阶段调用 LLM，报告内容可能在最后一刻被修改（引入幻觉、丢失证据链、改变结论措辞）。确定性渲染保证"研究阶段产出的内容就是最终报告的内容"。

## 9.2 三段式渲染流水线

```python
async def write_html_report(research_result, layout_plan):
    # 阶段1：结果归一化
    document_ir = await build_report_document(research_result, layout_plan)
    # 阶段2：HTML 渲染
    return await render_report_html(document_ir)
```

### 阶段 1：结果归一化（_normalize_research_result）

输入：`research_result`（可能字段缺失、格式不完全标准）

```python
def _normalize_research_result(research_result):
```

每个 section 的归一化填充了默认值（缺失 `body` → "本章节尚未提供正文内容"，缺失 `evidence_chain` → 空数组），保证下游渲染代码不会因为字段缺失而崩溃。

### 阶段 2：构建 Document IR

```python
async def build_report_document(research_result, layout_plan):
    return {
        "version": "deep-research-report-ir/v1",
        "title": ...,
        "subtitle": layout.get("subtitle"),
        "theme": layout.get("theme", "professional"),
        "executive_summary": ...,
        "sections": normalized_sections,
        "sources": normalized_sources,
    }
```

Document IR 是一个中间表示（Intermediate Representation）——介于原始研究数据和最终 HTML 之间的结构化对象。引入 IR 的好处：
- 如果将来要支持 PDF/Word 导出，只需新增渲染器，复用 IR
- IR 结构固定，方便单测验证

### 阶段 3：HTML 渲染

```python
async def render_report_html(document_ir):
    html = (
        "<!doctype html><html lang=\"zh-CN\"><head>"
        f"<title>{escape(document['title'])}</title>"
        f"<style>{_build_css()}</style>"
        "</head><body>"
        "<article class=\"report-paper\">"
        f"{_render_hero(document)}"
        f"{_render_toc(document['sections'])}"
        f"{_render_summary(document)}"
        "<div class=\"report-body\">"
        f"{''.join(_render_section(s) for s in document['sections'])}"
        "</div>"
        f"{_render_references(document['sources'], document['sections'])}"
        "</article>"
        "</body></html>"
    )
```

HTML 结构自上而下：
1. **Hero**：报告标题 + 副标题
2. **TOC**：可折叠树形目录
3. **Summary**：核心摘要卡片
4. **Sections**：所有章节正文
5. **References**：参考来源列表

## 9.3 概览章节 vs 叶子章节

```python
def _apply_section_roles(sections):
    parents = _compute_parent_sections(sections)  # 找出父章节
    for section in sections:
        section["is_overview"] = section["section_id"] in parents

def _render_section(section):
    if is_overview:
        # 父章节：只渲染标题 + 摘要 + 引言（截断到第一个子标题前）
        body_text = _truncate_at_first_subheading(body_text)
        return f"<h2>{title}</h2><p>{summary}</p>{body_md}"
    else:
        # 叶子章节：渲染完整结构
        return f"""
            <h2>{title}</h2>
            <p class="section-summary">{summary}</p>
            {body_md}
            {_render_key_findings(key_findings)}
            {_render_evidence_inline(evidence_chain)}
            {_render_table(...)}
            {_render_chart_placeholder(...)}
            {_render_risks(risks)}
        """
```

**父章节的识别**：如果一个 section 的 `section_id` 是其他 section_id 的前缀（如 "2" 是 "2.1" 和 "2.2" 的前缀），则它是父章节。

父章节只显示引言段落，不重复显示子章节的内容。`_truncate_at_first_subheading` 在第一个 `###` 或 `##` 处截断正文。

## 9.4 目录树构建算法

从平铺的 section 列表构建层级树：

```python
def _build_toc_tree(sections):
    all_ids = {sec.get("section_id") for sec in sections}
    node_map = {sid: {"section_id": sid, "title": ..., "children": []}
                for sid, sec in sections}

    for sid, node in node_map.items():
        parent_id = _parent_section_id(sid, all_ids)
        if parent_id and parent_id in node_map:
            node_map[parent_id]["children"].append(node)  # 挂到父节点
        else:
            roots.append(node)  # 没有父节点 → 根节点
    return roots

def _parent_section_id(section_id, all_ids):
    if "." not in section_id:
        return None
    parent = section_id.rsplit(".", 1)[0]  # "2.2.3" → "2.2"
    while parent:
        if parent in all_ids:
            return parent
        if "." not in parent:
            return None
        parent = parent.rsplit(".", 1)[0]  # 继续向上找
    return None
```

逐级向上查找父节点。例如 `section_id = "2.2.3"`：
1. 先找 `"2.2"` → 如果在 all_ids 中，它就是父节点
2. 如果 `"2.2"` 不在，继续向上找 `"2"`
3. 如果 `"2"` 也不在，该节点是根节点

然后递归渲染为 HTML `<details>/<summary>` 结构，纯 CSS 实现折叠/展开，无需 JavaScript。

### 纯 CSS 折叠

```html
<li class="toc-parent">
  <details open>
    <summary><a href="#section-2">2. 政策与市场</a></summary>
    <ul>
      <li class="toc-leaf"><a href="#section-2.1">2.1 政策环境</a></li>
      <li class="toc-leaf"><a href="#section-2.2">2.2 市场需求</a></li>
    </ul>
  </details>
</li>
```

`<details open>` 默认展开，`::before` 伪元素做三角形图标，CSS `transform: rotate(-90deg)` 做折叠动画。零依赖。

## 9.5 Markdown 正文渲染

```python
_md_parser = MarkdownIt("commonmark", {"typographer": True}).enable(["table", "strikethrough"])

def _render_body_markdown(text):
    stripped = text.strip()
    if not stripped:
        return "<p>本章节尚未提供正文内容。</p>"
    return _md_parser.render(stripped)
```

Agent 产出的 `body` 是 Markdown 格式。渲染时通过 `markdown-it-py` 转为 HTML。开启了 `table` 和 `strikethrough` 扩展。

### 首行标题去重

```python
def _strip_leading_heading(body):
    match = re.match(r"^#{1,6}\s+.+?(?:\s+#{1,6})?\s*\n", stripped)
    if match:
        return stripped[match.end():]
    return stripped
```

Agent 可能在 body 开头又写了一遍章节标题（如 `## 2.1 政策环境`）。但渲染层已经在外层 `<h2>` 中渲染了标题。`_strip_leading_heading` 去掉这行，避免重复。

## 9.6 证据链和引用渲染

### 行内引用

```python
def _render_evidence_inline(evidence_chain):
    all_source_ids = []
    for evidence in evidence_chain:
        for sid in evidence.get("source_ids", []):
            if sid not in all_source_ids:
                all_source_ids.append(sid)
    citations = _build_citations(all_source_ids)
    return f'<p class="evidence-inline"><span>证据来源：</span>{citations}</p>'
```

### 引用角标

```python
def _build_citations(source_ids):
    for source_id in source_ids:
        label = source_id.removeprefix("source-")  # "source-3" → "3"
        citations.append(
            f'<a href="#ref-{source_id}" class="cite-link">'
            f'<sup>[{label}]</sup></a>')
```

点击 `[3]` 跳转到页面底部的参考来源 `#ref-source-3`，目标来源高亮（`:target` 伪类）。

### 参考来源列表

```python
def _render_references(sources, sections):
    for source in sources:
        source_text = f"{source['title']}，{source['published_at']}"
        if source.get("url"):
            source_text += f'，<a href="{url}">{url}</a>'
        parts.append(f'<li id="ref-{source_id}">{source_text}</li>')
```

如果 sources 为空，从 evidence_chain 自建来源列表作为兜底。

报告整体风格偏专业简洁——类似咨询公司的研究报告。自包含（所有样式内嵌，无外部 CSS 依赖），HTML 可以直接在任何浏览器打开。

不同元素有独立的视觉区分：
- 核心摘要：浅绿背景卡片
- 关键发现：浅绿边框框
- 风险说明：浅黄背景警告框
- 父/概览章节：左侧绿色竖线
- 图表占位：浅绿框 + 描述文字

响应式断点在 640px，小于这个宽度报告纸张缩小边距。

## 9.8 设计取舍

| 决策 | 选择 | 原因 |
|------|------|------|
| 渲染方式 | 纯 Python 字符串拼接 | 不需要 Jinja2，减少依赖 |
| Markdown → HTML | markdown-it-py | 轻量、CommonMark 标准 |
| CSS 方案 | 自包含 `<style>` | HTML 可离线打开，不依赖 CDN |
| 图表 | 占位符（不实际渲染） | Agent 产出的 chart 数据不足以直接画图 |
| 目录折叠 | `<details>/<summary>` | 零 JS，纯 HTML+CSS |
| CSS 框架 | 手写 CSS | 报告是固定版式，不需要 Tailwind 的灵活性 |
