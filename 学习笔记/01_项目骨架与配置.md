## 1.2 配置管理（`app/config/config.py`）

### 核心机制：pydantic-settings BaseSettings

```python
class Settings(BaseSettings):
    app_name: str = "AI 研究报告工作台"
    mongodb_uri: str = "mongodb://localhost:27017"
    llm_temperature: float = Field(default=0.2, ge=0.0, le=2.0)
    tavily_api_key: str | None = None
    # ...

    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        extra="ignore",
    )
```

**自动映射规则**：
- 字段名 `mongodb_uri` → 环境变量 `MONGODB_URI`
- 字段名 `api_prefix` → 环境变量 `API_PREFIX`
- 优先级：环境变量 > `.env` 文件 > 代码默认值

### 关键设计点

**1. 空字符串转 None**

```python
def _empty_str_to_none(v: object) -> object:
    if isinstance(v, str) and v.strip() == "":
        return None
    return v

OptionalUrl = Annotated[AnyUrl | None, BeforeValidator(_empty_str_to_none)]
```

`.env` 文件中写了 `OPENAI_API_BASE=`（留空），pydantic 会把空字符串传给 `AnyUrl` 校验，导致报错。`BeforeValidator` 在类型校验前把空字符串转为 `None`，避免此问题。

**2. 单例缓存**

```python
@lru_cache(maxsize=1)
def get_settings() -> Settings:
    return Settings()
```

`@lru_cache(maxsize=1)` 保证整个进程生命周期中只实例化一次 Settings，只读一次 `.env` 文件。所有模块都通过 `get_settings()` 获取同一个配置对象，避免重复 IO。

**3. Field 校验**

```python
llm_temperature: float = Field(default=0.2, ge=0.0, le=2.0)
```

`ge=0.0, le=2.0` 是 pydantic 的内置校验，温度值超出范围时启动阶段就会报错（fail-fast），不会等到运行时才发现。

### 配置项分类

| 分类 | 字段 | 说明 |
|------|------|------|
| 应用基础 | `app_name`, `app_version`, `log_level` | FastAPI 文档标题和日志级别 |
| 数据存储 | `mongodb_uri`, `mongodb_database`, `redis_url` | MongoDB 连接和数据库名 |
| LLM | `llm_provider`, `llm_model_name`, `llm_temperature` | 模型选择，provider 为 `openai` 或 `deepseek` |
| LLM 密钥 | `openai_api_key`, `deepseek_api_key`, `openai_api_base` | 可选，按 provider 选择使用 |
| RAGFlow | `ragflow_base_url`, `ragflow_api_key`, `enable_ragflow` | 内部知识库检索 |
| 外部搜索 | `tavily_api_key` | Tavily 搜索 API |
| 对象存储 | `object_storage_*`, `report_storage_backend` | 报告 HTML 存储（local/minio） |

## 1.3 应用入口（`app/main.py`）

### 工厂函数模式

```python
def create_app() -> FastAPI:
    settings = get_settings()
    app = FastAPI(
        title=settings.app_name,
        version=settings.app_version,
        docs_url=f"{settings.api_prefix}/docs",        # /api/v1/docs
        openapi_url=f"{settings.api_prefix}/openapi.json", # /api/v1/openapi.json
    )
```

使用工厂函数而非模块级 `app = FastAPI()` 的好处：
- 测试时可以创建独立实例
- 配置注入更清晰
- 避免模块导入时的副作用

但当前项目最终还是创建了模块级实例：
```python
app: FastAPI = create_app()  # uvicorn app.main:app 用的就是这个
```

### 中间件注册顺序（重要）

```python
# 1. CORS —— 最先注册
app.add_middleware(CORSMiddleware, allow_origins=["*"], ...)

# 2. API 路由 —— 次优先
app.include_router(research_projects_router, prefix=settings.api_prefix)

# 3. 静态文件 —— 最后挂载
app.mount("/", StaticFiles(directory="static", html=True), name="static")
```

**为什么顺序很重要**：FastAPI 按注册顺序匹配路由。静态文件 `mount("/", ...)` 会匹配所有路径。如果先注册静态文件，`GET /api/v1/research-projects/xxx` 会被静态文件处理器拦截，返回 404。

把 `mount` 放最后，FastAPI 先尝试匹配 API 路由，匹配不到才 fallback 到静态文件（SPA 的 `html=True` 会让所有未匹配路径返回 `index.html`）。

### 健康检查端点

```python
@app.get("/health", tags=["系统"])
async def health_check() -> dict[str, str]:
    return {"status": "ok"}
```

在静态文件挂载之前注册，确保 `/health` 不被 SPA fallback 吃掉。用于 Docker 健康探针或负载均衡器的存活检查。

## 1.5 关键设计决策

1. **单进程异步，无消息队列**：后台任务用 `asyncio.create_task` 而非 Celery，牺牲了水平扩展能力，但换来了极简的部署和调试体验。适合企业内部小规模使用。

2. **所有外部服务可选**：Tavily 未配置 → 搜索返回 `skipped`，RAGFlow 未配置 → 跳过内部检索。系统在任何时候都有降级路径，不会因为缺少某个外部依赖而崩溃。

3. **配置统一入口**：整个项目只有 `get_settings()` 一个配置来源，不存在散落的 `os.getenv()` 调用。类型安全、可追溯、可测试。

4. **占位模式**：Agent 未接入时，系统仍能返回结构完整的占位数据。这让前后端可以独立开发联调。
